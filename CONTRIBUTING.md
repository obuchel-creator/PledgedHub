# Contributing to PledgeHub

## Spec-Driven Development Process

- All features and changes must reference product-spec.md and technical-plan.md.
- Before coding, review the relevant spec and update if requirements change.
- Implementation must strictly follow the order in tasks.md.
- All code must be validated against the specs before merging.
- Automated tests should be derived from acceptance criteria in product-spec.md.
- PRs must include a checklist showing spec compliance.

## How to Contribute
1. Read product-spec.md and technical-plan.md.
2. Pick the next uncompleted task from tasks.md.
3. Implement with production-ready code and comments.
4. Write tests based on the spec.
5. Submit PR with spec compliance checklist.
